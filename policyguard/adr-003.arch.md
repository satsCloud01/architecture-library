# ADR-003 — AI-Assisted Policy Authoring

**Status:** Accepted
**Date:** 2025-01
**Author:** Platform Team

---

## Context

Writing ABAC policy JSON from scratch requires deep familiarity with the policy schema. Compliance officers and data stewards without engineering backgrounds find it difficult to:
1. Know which subject/resource/action JSON fields are available
2. Translate a natural-language governance rule into structured JSON
3. Identify whether a new policy conflicts with an existing one

We want to reduce this barrier by allowing users to describe a rule in plain English and have an AI model produce a valid policy JSON suggestion that can be reviewed and committed.

---

## Decision

### Model: Claude Haiku (`claude-haiku-4-5-20251001`)

Claude Haiku is used for three AI features in Policy Studio:

#### 1. Policy Suggestion (`POST /api/ai/suggest`)

**Input:** `{ "description": "Only EU data scientists with clearance ≥ 2 can SELECT from the customers table" }`

**System prompt strategy:**
- Provide the full policy JSON schema with field types, allowed values, and examples
- Instruct the model to return only a JSON object (no markdown, no explanation)
- Set `temperature: 0` for deterministic structured output

**Output:** A partial or full policy JSON ready to pre-fill the Policy Studio form

#### 2. Conflict Detection (`POST /api/ai/conflicts`)

**Input:** Draft policy JSON + list of names/summaries of active policies (not full JSON — avoids token bloat)

**System prompt strategy:**
- Ask Claude to identify overlapping rules (same datasource/action/subject combination)
- Return a structured list of `{ conflicting_policy_id, reason }` objects

#### 3. Impact Analysis (`POST /api/ai/impact`)

**Input:** Policy JSON change (before/after) + audit stats for affected datasource

**System prompt strategy:**
- Ask Claude to describe in 2–3 sentences what requestors will be newly permitted or denied
- Return a human-readable narrative (no JSON required)

### Key Design Choices

**API key stored in `settings` table, not environment variable:**
- Allows non-technical users to configure their own key via the Settings UI
- No server restart required when key is updated

**Graceful degradation when key absent:**
- `POST /api/ai/suggest` returns `{ "suggestion": null, "message": "Add your Anthropic API key in Settings to enable AI features" }`
- Policy Studio UI hides the AI Assist panel if suggestion is null

**No PII in prompts:**
- Policy metadata (field names, roles, datasource names) is structural data, not customer PII
- Prompts never include actual data from the protected datasources

---

## Consequences

**Positive:**
- Lowers time-to-policy from minutes to seconds for non-technical governance stakeholders
- Claude's structured JSON output quality is high for schema-constrained tasks at `temperature: 0`
- Conflict detection catches common mistakes (overlapping deny + permit on same datasource) before activation
- AI features are fully optional — the system works completely without an API key

**Negative / Trade-offs:**
- Requires an outbound HTTPS connection to `api.anthropic.com`; not suitable for air-gapped deployments as-is
- Haiku suggestions should always be reviewed by a human before activating a policy — the system enforces this via the `draft` → `review` → `active` state machine
- Cost: each suggestion call is ~500–1000 input tokens + ~300 output tokens ≈ $0.0001/call (negligible)

---

## Alternatives Considered

### Fine-tuned local model (LLaMA / Mistral)

Rejected because:
- Would require GPU-enabled infrastructure for acceptable inference speed
- Fine-tuning dataset curation is expensive
- Claude Haiku at `temperature: 0` produces better structured JSON than open-source 7B models without fine-tuning

### GPT-4o / GPT-3.5-turbo (OpenAI)

Rejected because:
- Team preference for Anthropic models (consistent with other projects in the portfolio)
- Claude Haiku structured JSON compliance tested to be superior for our schema

### Rule-based template wizard (no AI)

Considered as a fallback — a step-by-step form wizard that builds JSON from dropdowns. Retained as the primary UI (Policy Studio form fields). AI Assist is an additive layer on top, not a replacement.

---

## Review

This ADR will be revisited if:
- Policy volume exceeds 500 active policies and conflict detection prompts exceed Claude's context window
- Air-gapped deployment becomes a hard requirement (triggers local model evaluation)
- Anthropic API pricing changes materially impact usage economics
