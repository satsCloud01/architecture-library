# ADR-001 — Technology Stack Selection

**Status:** Accepted
**Date:** 2025-01
**Author:** Platform Team

---

## Context

We are building the Dynamic Policy Access Engine (PolicyGuard) — a full-stack ABAC policy management and runtime evaluation system. The system must support:

- A REST API capable of evaluating access decisions with sub-10ms latency
- A rich admin UI for policy authoring, simulation, and audit review
- AI-assisted policy authoring via Claude
- A local-first, zero-infrastructure development experience
- Easy single-server production deployment on a t2.micro EC2 instance

The team is experienced with Python for backend and React/TypeScript for frontend.

---

## Decision

### Backend: FastAPI + Python 3.12

**FastAPI** was chosen over Django REST Framework and Flask because:

1. **Async-native**: `async/await` throughout — critical for non-blocking ABAC evaluation under concurrent load
2. **Pydantic v2 integration**: Request/response models declared as Python dataclasses; automatic validation and serialisation
3. **OpenAPI auto-generation**: `/docs` endpoint provides an interactive playground for all ABAC scenarios
4. **Performance**: ASGI + uvicorn outperforms WSGI stacks (Flask, Django) at equivalent concurrency
5. **Startup speed**: No Django ORM overhead; minimal cold-start latency

**Python 3.12** for:
- Pattern matching (used in combining algorithm dispatch)
- Latest `asyncio` improvements
- Active security patch cadence

### ORM: SQLAlchemy 2.x (async)

SQLAlchemy 2.x async was chosen over raw `aiosqlite` and over Tortoise ORM because:
- Mature, battle-tested ORM with excellent typing support
- `async_sessionmaker` pattern fits FastAPI's dependency injection cleanly
- Seamless swap between SQLite (dev/prod single-server) and PostgreSQL (future scale-out)

### Database: SQLite → PostgreSQL migration path

**SQLite** in Phase 1 (single-server):
- Zero infrastructure: no Docker, no RDS, no local Postgres required for development
- `aiosqlite` provides async driver compatible with SQLAlchemy 2.x
- Sufficient for < 1M audit log rows at projected demo/pilot load

**PostgreSQL** when needed (Phase 2):
- Swap `DATABASE_URL` env var; no ORM code changes required
- Enables read replicas for audit log analytics

### Frontend: React 18 + Vite + TypeScript + Tailwind CSS

**React 18** for:
- Concurrent rendering for smooth chart updates (Recharts live data)
- `useTransition` for non-blocking state updates during filter operations
- Large ecosystem (React Router, Recharts, lucide-react)

**Vite** over Create React App:
- ~10× faster cold HMR in development
- Native ESM; smaller production bundles
- Built-in TypeScript support

**Tailwind CSS** over Material UI / Chakra UI:
- No component library opinion lock-in
- Dark theme as first-class citizen (slate-900 design system)
- Smaller bundle: only used utility classes are included

**Recharts** for data visualisation:
- Composable SVG-based charts that integrate cleanly with React state
- Responsive container support out of the box
- Better TypeScript types than Chart.js alternatives

### AI: Anthropic Claude Haiku

**Claude Haiku** was chosen for AI-assisted policy authoring over GPT-3.5-turbo and local LLMs:
- Cost: lowest cost-per-token in the Claude family; policy suggestion calls are infrequent
- Quality: structured JSON output reliability is higher than GPT-3.5-turbo for schema-constrained tasks
- Security: no PII leaves the system (policy metadata is not customer data)
- Fallback: graceful degradation when API key is absent (feature hidden, not crashed)

---

## Consequences

**Positive:**
- Single `uvicorn` process + SQLite file = trivial deployment on any Linux VM
- Full TypeScript type safety from API response to React component props
- Pydantic models serve as single source of truth for request/response schemas
- ABAC engine is a pure function (no I/O) — trivially unit-testable

**Negative / Trade-offs:**
- SQLite has a single-writer limit; concurrent write-heavy workloads will require migration to PostgreSQL (acceptable for current pilot scope)
- No native horizontal scaling with SQLite; must migrate to PostgreSQL + connection pooler for multi-instance deployment
- Claude Haiku AI features require an Anthropic API key; not suitable for air-gapped deployments without modification

---

## Alternatives Considered

| Option | Rejected Because |
|---|---|
| Django REST Framework | Heavier, synchronous-first, overkill for single-domain API |
| Node.js / Express | Team Python expertise; Python ecosystem for future ML integration |
| PostgreSQL from day 1 | Adds infrastructure complexity for local dev; SQLite provides same API surface via SQLAlchemy |
| GPT-3.5-turbo | Structured JSON compliance weaker; higher cost for same quality |
| shadcn/ui component library | Adds Radix dependency; Tailwind utilities sufficient for this scope |
