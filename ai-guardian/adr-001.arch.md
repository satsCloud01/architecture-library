# ADR-001: Technology Stack Selection

**Status:** Accepted
**Date:** 2025-03-01
**Authors:** Platform Engineering Team

---

## Context

Enterprise AI Guardian requires a full-stack web application capable of:
- Serving a rich interactive UI for governance workflows
- Providing a structured REST API for data access
- Persisting governance data with relational integrity
- Supporting rapid development for a pitch/demo context with a clear upgrade path to production

---

## Decision

### Backend: FastAPI + Python 3.12 + SQLAlchemy async

**Rationale:**
- FastAPI provides automatic OpenAPI documentation, Pydantic v2 validation, and async-first design
- Python 3.12 aligns with the organisation's existing ML/data science tooling
- SQLAlchemy 2.0 async with `aiosqlite` enables non-blocking DB operations
- `greenlet` included as required for SQLAlchemy async on macOS environments

### Frontend: React 18 + TypeScript + Vite + Tailwind CSS

**Rationale:**
- React 18 with hooks provides a mature component model and large ecosystem
- TypeScript ensures type safety across the API client and component props
- Vite provides sub-second HMR and faster builds than webpack/CRA
- Tailwind CSS eliminates the need for a component library while providing design consistency
- Recharts (built on D3/SVG) provides lightweight charting without heavy dependencies

### Database: SQLite (demo) → PostgreSQL (production)

**Rationale:**
- SQLite requires zero infrastructure for the demo context
- SQLAlchemy async supports both `aiosqlite` and `asyncpg` with identical query code
- Switching to PostgreSQL in production requires only a connection string change

### Seed Data: Python script (idempotent)

**Rationale:**
- Allows reproducible demo data without a separate migration tool
- Idempotency check (count > 0) prevents duplicate seeding on restart
- Faker library available for extending seed data if needed

---

## Consequences

**Positive:**
- Backend is self-documenting via `/docs` and `/redoc`
- Frontend build is fully static — can be deployed to any CDN
- Single language (Python) for backend reduces cognitive overhead
- TypeScript types in `lib/api.ts` mirror backend Pydantic schemas

**Negative:**
- SQLite does not support concurrent writes — acceptable for demo, must switch for production
- No authentication implemented in demo — must add JWT middleware before production
- Recharts has limited customisation compared to Victory or Nivo

---

## Alternatives Considered

| Alternative | Rejected Because |
|---|---|
| Django + DRF | Heavier ORM, slower async support, more boilerplate |
| Express.js backend | Team's primary language is Python |
| Vue / Svelte | React team familiarity |
| MongoDB | Relational integrity required for governance data |
| shadcn/ui | Requires interactive `npx` install; Tailwind utilities sufficient |
